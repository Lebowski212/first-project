Git — это система контроля версий. Ниже простой, цельный текст, как ты реально будешь с ним работать, без списков и теории.

Сначала зайди в папку с проектом и проверь, что ты находишься там, где нужно. Для этого используются команды pwd и ls. После этого инициализируй репозиторий командой git init. В этот момент Git начинает отслеживать проект.

Чтобы понять текущее состояние файлов, используется команда git status. Она показывает, какие файлы изменены, какие добавлены, а какие ещё нет. После любых изменений файлы нужно добавить в индекс. Обычно добавляют всё сразу командой git add ., но можно добавить и один конкретный файл через git add имя_файла.

Когда файлы добавлены, необходимо зафиксировать изменения. Это делается командой git commit -m "описание изменений". Если ты не сделал коммит, значит ты ничего не сохранил — изменения существуют только у тебя локально. Историю коммитов можно посмотреть через git log, а если нужна короткая версия — git log --oneline.

Чтобы связать локальный проект с удалённым репозиторием на GitHub или GitLab, используется команда git remote add origin ссылка_на_репозиторий. Проверить, что связь установлена, можно командой git remote -v. Перед первой отправкой кода обычно переименовывают ветку в main командой git branch -M main, после чего отправляют проект на сервер командой git push -u origin main. В дальнейшем достаточно писать просто git push.

Если нужно получить изменения из удалённого репозитория, используется команда git pull. Если же репозиторий уже существует и его нужно скачать на компьютер, применяется команда git clone ссылка_на_репозиторий.

Иногда возникает необходимость убрать файл из Git, но оставить его на компьютере. Для этого используется команда git rm --cached имя_файла. Если ты добавил файлы через git add, но передумал, можно отменить это командой git reset. Текущие ветки проекта можно посмотреть командой git branch.

Рабочая схема всегда одна и та же: ты изменяешь файлы, затем добавляешь их через git add, потом фиксируешь изменения командой git commit, и только после этого отправляешь всё на сервер через git push. Всё остальное — вторично. Если коммитов нет, Git для тебя бесполезен.
